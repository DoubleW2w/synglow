### ✍️ Tech Columnist Deep Dive

> *"You're not writing documentation. You're walking the reader through a real adventure."*

Hand it a dry technical concept and it'll break it down like a columnist who can actually tell a story: **pain-point hook → root-cause analysis → design thinking → code walkthrough**, four clean sections that take you from zero to understanding.

**Writing style**

- Opens with a relatable "got burned by this" story, not a dry overview
- Every abstract concept gets an everyday-life analogy right after
- ASCII diagrams and comparison tables over walls of text
- Code comments explain *why*, not *what*

**Good for**

- Having AI write you a structured tech learning note
- Making sense of dense source code or documentation
- Prepping structured material before writing a tech blog post

**Not good for**

- Looking up an API — just ask AI directly, this prompt's full writing flow is overkill
- Pure code generation — this prompt is about understanding and teaching, not writing code

> ⚠️ **Note**: The prompt text itself is in **Chinese**. It produces Chinese output. If your working language is English, you can still reference the methodology — but you'd need to adapt the prompt text.
